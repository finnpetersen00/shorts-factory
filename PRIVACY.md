# Privacy Policy — Shorts Factory

_Last updated: 9 July 2026_

Shorts Factory is a personal, single-user desktop application operated by
its developer. It is not distributed to, installed by, or usable by anyone
other than the developer. It has no user accounts, no website login, and
no third-party users whose data could be collected.

## What the application does

Shorts Factory generates short videos on the developer's own computer and
uploads them to the developer's own YouTube channel using the YouTube
Data API v3 (`youtube.upload` scope only).

## YouTube API Services

This application uses YouTube API Services. By using it, the developer
(its only user) agrees to be bound by the
[YouTube Terms of Service](https://www.youtube.com/t/terms). Google's
handling of information is described in the
[Google Privacy Policy](http://www.google.com/policies/privacy).

## Data accessed, collected, and stored

- The application accesses only the developer's own YouTube account,
  authorized via Google OAuth 2.0.
- The OAuth token is stored locally on the developer's computer and is
  never transmitted anywhere except to Google's own OAuth and YouTube API
  endpoints.
- The application does not access, collect, store, or share any data
  belonging to any other person. It does not read comments, subscriber
  lists, analytics, or any other channel's data.
- No API data is shared with, sold to, or processed by any third party.
- No cookies are used and no advertising or analytics services are
  embedded.

## Revoking access

Access can be revoked at any time via
[Google security settings](https://security.google.com/settings/security/permissions),
which immediately invalidates the stored token.

## Contact

For any questions about this policy, contact the developer via the
repository's issue tracker.
